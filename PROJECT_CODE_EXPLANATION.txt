                    SMART VILLAGE MART - PROJECT CODE EXPLANATION
================================================================================

PROJECT OVERVIEW
================================================================================
Smart Village Mart is a full-stack e-commerce platform designed to connect 
local village shopkeepers with customers. The platform enables shopkeepers to 
list their products online and allows customers to browse, search, and purchase 
authentic local products.

TECHNOLOGY STACK
================================================================================
Frontend:
- React 18.3.1 with TypeScript
- Vite (Build tool)
- React Router DOM (Navigation)
- Tailwind CSS 4.1.12 (Styling)
- Radix UI (Component library)
- Lucide React (Icons)
- React Helmet Async (SEO)
- Axios (HTTP client)

Backend:
- Node.js with Express 5.2.1
- MySQL 2 (Database)
- JWT (Authentication)
- Bcrypt (Password hashing)
- CORS (Cross-origin requests)

3. AUTHENTICATION CONTEXT - AuthContext.tsx
================================================================================
Location: src/contexts/AuthContext.tsx

Purpose: Manages user authentication state across the entire application

Code Explanation:

A. State Management:
   - user: Stores current user information (id, name, email, role)
   - loading: Tracks authentication check status
   - isAuthenticated: Boolean flag for login status

B. Key Functions:

   login(email, password, role):
   - Sends POST request to backend with credentials
   - Receives JWT token and user data
   - Stores token in localStorage
   - Updates user state
   - Returns success/error

   logout():
   - Removes token from localStorage
   - Clears user state
   - Redirects to home page

   checkAuth():
   - Runs on app initialization
   - Checks if token exists in localStorage
   - Validates token with backend
   - Restores user session if valid

C. Context Provider:
   - Wraps entire app to provide auth state and functions
   - Available to all components via useAuth() hook

17. AUTHENTICATION FLOW
================================================================================

A. Login Process:
   1. User enters email and password
   2. Frontend sends POST request to /api/auth/{role}/login
   3. Backend queries database for user
   4. Backend compares password hash with bcrypt
   5. If valid, backend generates JWT token
   6. Token contains: user id, email, role
   7. Frontend stores token in localStorage
   8. Frontend updates AuthContext state
   9. User is redirected to dashboard

B. Protected Routes:
   1. User tries to access protected route
   2. ProtectedRoute component checks AuthContext
   3. If not authenticated, redirect to home
   4. If authenticated but wrong role, redirect to home
   5. If authorized, render the component

C. Token Verification:
   1. On app load, checkAuth() runs
   2. Checks if token exists in localStorage
   3. Sends token to /api/auth/verify
   4. Backend verifies JWT signature
   5. If valid, returns user data
   6. Frontend restores user session
   7. If invalid, clears localStorage

D. Logout Process:
   1. User clicks logout
   2. Frontend removes token from localStorage
   3. Frontend clears AuthContext state
   4. User redirected to home page

================================================================================
                        TROUBLESHOOTING
================================================================================

25. COMMON ISSUES
================================================================================

A. Database Connection Error:
   - Check MySQL is running
   - Verify .env credentials
   - Check database exists

B. CORS Error:
   - Verify backend CORS configuration
   - Check frontend API URL
   - Ensure credentials: true

C. Authentication Issues:
   - Check JWT secret matches
   - Verify token in localStorage
   - Check token expiration

D. Image Upload Fails:
   - Check uploads folder exists
   - Verify folder permissions
   - Check file size limits

E. Build Errors:
   - Clear node_modules
   - Delete package-lock.json
   - Run npm install again
